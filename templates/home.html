Stock Tracker API

A Django-based API for inventory management.

About This Project

This API provides the necessary backend functionality for a stock management system. It allows users to manage a product catalog and track inventory movements through stock-in (receiving) and stock-out (shipping) transactions. The system is designed with a clean API structure, making it easy to integrate with a frontend application.

How to Use the API

Follow this workflow to test the core features. All endpoints can be tested directly in the browser via the browsable API.

Step 1: Create a Product

First, add a product to the master list. Navigate to the /api/products/ endpoint and use the HTML form to POST a new product.

  • GTIN: A unique 13 or 14-digit identifier. Example: 5012345678900
  • Product Name: The name of the product. Example: Standard Widget

Step 2: Add Stock (Stock In)

Next, record that you have received inventory. Go to the /api/stock/in/ endpoint and POST a JSON object in the "Content" text area.

Example JSON Payload:

{
    "items": [
        {
            "product_gtin": "5012345678900",
            "quantity": 100
        }
    ]
}

Step 3: Check Stock Level

To see the current inventory for any product, navigate to the /api/stock/level/<gtin>/ URL in your browser, replacing <gtin> with your product's GTIN.

Example URL: /api/stock/level/5012345678900/

Step 4: Remove Stock (Stock Out)

Finally, to record a shipment, go to the /api/stock/out/ endpoint and POST a JSON object, similar to a stock-in transaction.

Example JSON Payload:

{
    "items": [
        {
            "product_gtin": "5012345678900",
            "quantity": 15
        }
    ]
}

API Endpoint Summary

  • /api/products/ (GET, POST) - View all products or create a new one.
  • /api/stock/in/ (POST) - Record incoming inventory.
  • /api/stock/out/ (POST) - Record outgoing inventory.
  • /api/stock/level/<gtin>/ (GET) - Get the current stock level for a specific product.
